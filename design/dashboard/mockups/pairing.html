Why a PIN? Your network identity got you here, but identity alone doesn't grant control. The PIN proves you can see the host machine's terminal — so a stolen or shared allowlist entry can't quietly gain access.

What pairing grants. This browser gets a signed cookie for this host. It can chat with agents, run commands through them, browse project files, and upload/download — the same power as sitting at the terminal. Unpair anytime from settings → devices.
